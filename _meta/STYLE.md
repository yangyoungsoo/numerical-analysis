# 노트북 스타일 가이드

매일 자동 생성되는 ipynb 노트북은 **Day01/CE_1_1_01.ipynb** 의 구조를 따른다. 핵심 원칙:

## 셀 구성 (순서대로)

1. **제목 + 메타** (markdown)
   - `# Computer Exercise X.Y — Problem Z`
   - blockquote으로 교재, 단원, 풀이 일자, 언어 표기.

2. **문제 (원문)** (markdown)
   - `## 1. 문제 (원문)` 헤더
   - 영문 원문 그대로 인용 (`> ...`)
   - `### 한국어 풀이용 정리` 서브섹션으로 한국어 요약

3. **수학적 배경** (markdown)
   - `## 2. 수학적 배경` 헤더
   - 정의, 정리, 오차 분석 등 LaTeX 수식 사용
   - 수식은 inline은 `$...$`, display는 `$$...$$`
   - 굵은 강조는 핵심 결과 boxed: `\boxed{ ... }`

4. **풀이 흐름** (markdown)
   - `## 3. 풀이 흐름` 헤더
   - 번호 매긴 단계별 설명 (보통 5–8단계)
   - 어떤 함수를 쓰는지, 어떤 입력으로, 무엇을 출력하는지

5. **코드 셀들** (code)
   - import: `numpy as np`, `pandas as pd`, `matplotlib.pyplot as plt`
   - 셀 1: 데이터/계산
   - 셀 2: 표 출력
   - 셀 3: 시각화
   - 그래프 라벨은 **영문** (한글 폰트 의존성 회피), 본문은 한국어

6. **결과 해석** (markdown)
   - `## 4. 결과 해석` 헤더
   - 그래프에서 읽어내야 할 점들을 번호로
   - **결론** 블록 (`> ...`) 한 줄로 핵심 메시지
   - 마지막에 **다음 문제 / 다음 단원으로의 연결** 추가

## 스타일 약속

* 한국어 본문, 영문 그래프 라벨
* 수식은 LaTeX, 모든 변수에 의미 부여
* 결과 해석은 *왜* 이렇게 나오는지에 집중 (현상 → 원인 → 의미)
* 코드는 실행 가능하게 작성. nbconvert로 `--execute` 가능해야 함
* 외부 데이터 의존하지 않음 (전부 코드 안에서 생성)
