# Day 61 — §14.5 Monitoring & Drift Detection (모니터링·드리프트 감지: 언제 다시 손보나)

Day60(§14.4)에서 파이프라인을 새 도메인으로 **전이**했다. 그러나 운영 중 target 분포는 *시간에 따라* 계속 변한다.
§14.5는 그 변화를 **감지**하고, 감지 신호로부터 **언제 재선택·재보정·재튜닝을 촉발할지**의 기준을 세운다.

$$\text{전이된 파이프라인}\;\xrightarrow{\;\text{모니터링}\;}\;\text{드리프트 감지}\;\xrightarrow{\;\text{트리거}\;}\;\text{재보정/재학습}.$$

> **공통 축**: 드리프트는 두 얼굴을 가진다 — 입력분포 $p(x)$ 가 변하는 **데이터(공변량) 드리프트**와,
> 관계 $p(y\mid x)$ 가 변하는 **개념 드리프트**. 전자는 *입력*으로, 후자는 *잔차*로 드러난다.
> 관통하는 뿌리는 여전히 **편향-분산 트레이드오프**와 **불확실성의 정량화**, 그리고 새로 더해진 **비용-정확 상충**이다.

## 학습 목표
- 고정 참조창 대비 **KS 통계량·PSI** 로 공변량 드리프트를 정량 감지하고, **지속성 규칙**으로 단발 스파이크(오경보)를 거른다.
- 입력이 불변이어도 나타나는 개념 드리프트를 **예측 잔차**의 **Page-Hinkley·CUSUM** 순차검정으로 잡고, **감지지연 ↔ 오경보** 곡선을 그린다.
- 두 감지기를 하나의 **트리거 정책**으로 묶어 frozen/periodic/reactive를 비교하고, 반응형이 **낮은 비용으로 높은 정확도**를 냄을 보인다.

## 한 줄 정리
> **모니터링 = "언제 손잡이를 다시 돌릴지 정하기"** — 공변량은 입력분포(KS/PSI)로, 개념은 잔차(PH/CUSUM)로 감지하고,
> 두 신호를 묶은 **반응형 트리거**로 *필요할 때만* 재보정해 주기적 유지의 정확도를 낮은 비용에 재현한다.
