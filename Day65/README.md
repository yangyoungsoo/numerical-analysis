# Day 65 — §14.9 Learning-Rate Schedules (학습률 스케줄 3부작: 언제·얼마나·다시 크게)

Day64(§14.8)에서 학습률 손잡이를 **좌표별로 자동 조정**하는 적응법(AdaGrad→RMSProp→Adam)을 다뤘다.
거기서도 남는 질문은 "**하나의 손잡이를 시간축 위에서 어떻게 움직일까**"였다. §14.9 는 그 시간축 스케줄을
세 각도로 본다 — **감쇠**(잡음 바닥 낮추기), **워밍업+담금질**(초기 안정+부드러운 착륙),
**웜 리스타트**(드리프트 추적).

$$\text{상수 }\eta \ \xrightarrow{\text{감쇠}}\ \text{step / }1/\sqrt t
\ \xrightarrow{+\text{초기 램프}}\ \text{warmup+cosine}
\ \xrightarrow{+\text{주기적 재설정}}\ \text{SGDR}.$$

> **공통 축**: §14.7–14.8 과 똑같이 **편향-분산**이다. 큰 $\eta$ = 빠르지만 잡음 큰(분산) 스텝,
> 작은 $\eta$ = 느리지만 정밀한(편향) 스텝. 스케줄이란 이 손잡이를 **시간에 걸쳐** 조율하는 일이다.

## 학습 목표
- 상수 학습률의 **잡음 바닥**($\propto\sqrt\eta$)을 확인하고, **step decay·$1/\sqrt t$** 감쇠가 "초반 크게·후반 작게"로
  빠른 하강과 낮은 최종오차를 **동시에** 얻음을 보인다.
- 공격적 peak 학습률이 만드는 **초기 오버슈트**를 **선형 워밍업**이 어떻게 제거하는지 정량화하고,
  **코사인 담금질**이 낮은 tail 을 주는 메커니즘을 이해한다(정규화 LMS 로 안정범위 $(0,2)$ 확보).
- 단조 감쇠가 **드리프트** 목표에서 추적에 실패함을 보이고, **코사인 웜 리스타트(SGDR)** 가 학습률을 주기적으로
  되살려 추적을 회복함을 확인하며, 재시작 주기 $T_c$ 를 **추적속도↔잡음** 편향-분산 손잡이로 해석한다.

## 한 줄 정리
> **학습률 스케줄 = "손잡이를 시간축 위에서 조율"** — 감쇠(잡음 바닥↓)·워밍업+코사인(초기 안정+정밀 착륙)·
> 웜 리스타트(드리프트 추적)는 모두 **하나의 편향-분산 이야기**이며, 정상 문제엔 단조 감쇠가·비정상 문제엔
> 웜 리스타트가 견고하다.
