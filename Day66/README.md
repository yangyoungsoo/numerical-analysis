# Day 66 — §14.10 Automated Hyperparameter Tuning (하이퍼파라미터 자동 튜닝: 손잡이 조율 자체를 자동화)

Day63–65(§14.7–14.9)에서 학습률 손잡이를 **좌표별**(적응법)·**시간축**(스케줄)으로 조율했다.
그러나 그 손잡이의 **값**과 **스케줄 형태**는 여전히 사람이 골랐다. §14.10 은 마지막 한 걸음 —
**손잡이 조율 자체를 자동화**한다. 세 각도로 본다: **범위 탐색**(한 번의 스윕으로 안정 상한 찾기),
**진화**(드리프트를 따라 온라인으로 학습률 재편), **예산 배분**(고정 예산을 후보들에 적응적으로 분배).

$$\text{수동 }\eta \ \xrightarrow{\text{한 번의 스윕}}\ \eta_{\max}\text{ 자동}
\ \xrightarrow{\text{온라인 진화}}\ \text{PBT}
\ \xrightarrow{\text{적응 예산}}\ \text{Successive Halving / Hyperband}.$$

> **공통 축**: §14.7–14.9 와 똑같이 **편향-분산**이다. 자동 튜닝은 결국 U자의 좋은 지점을
> **사람의 반복 실행 없이** 찾아내는 세 가지 방법이며, 정상 문제엔 범위 탐색·SH 가,
> 비정상(드리프트) 문제엔 PBT 가 자연스럽다.

## 학습 목표
- **학습률 범위 탐색(LR range test)** 을 구현해, 지수 스윕 한 번으로 *loss-vs-$\eta$* 곡선의 네 국면(평탄·급강하·최소·발산)을 얻고 **안정 상한 $\eta_{\max}=\eta_{\text{div}}/10$** 를 자동 추출한다. 정상 스트림에서 산출물은 "상수율"이 아니라 "$\eta_{\max}$"이며, 이를 §14.9 감쇠 스케줄의 peak 로 써서 **빠른 하강 + 낮은 바닥**을 동시에 얻음을 보인다.
- **Population-Based Training(PBT)** 으로 $K$개 온라인 학습기의 학습률을 주기적 **exploit(상태·$\eta$ 복사) + explore($\eta$ 섭동)** 로 진화시켜, 드리프트로 **이동하는 최적 $\eta^\star_t$** 를 온라인으로 추적하고 best-fixed 를 능가함을 확인한다.
- **Successive Halving / Hyperband** 로 **고정 예산**을 후보들에 적응적으로 배분해, 나쁜 후보를 초기 rung 에서 싸게 걸러내고 오라클 전수탐색 대비 **약 15% 예산**으로 near-oracle $\eta$ 를 선택함을 보인다. 감축인자 $\eta_{\text{sh}}$ 의 탐색-활용 상충을 해석한다.

## 한 줄 정리
> **자동 튜닝 = "손잡이 조율을 사람 손에서 알고리즘 손으로"** — 범위 탐색(한 스윕으로 안정 상한)·
> PBT(드리프트 추적하는 온라인 진화)·SH/Hyperband(적응 예산 배분)는 모두 **하나의 편향-분산 U자**의
> 좋은 지점을 반복 실행 없이 찾는 방법이며, 정상 문제엔 범위 탐색·SH, 비정상 문제엔 PBT 가 견고하다.
